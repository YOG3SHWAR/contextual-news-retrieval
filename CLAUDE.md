## Project Overview

This is a **Contextual News Data Retrieval System** — a backend take-home task for an Inshorts Software Engineer role. The goal is to build a REST API that retrieves news articles from `news_data.json`, processes queries using an LLM to extract entities/intent, and enriches results with LLM-generated summaries.

The JD specifies **Go or Java** as the preferred language, with optional Python. Docker/Kubernetes and Redis caching are preferred.

## Data Source

`news_data.json` — 2000 news articles. Schema:
```json
{
  "id": "uuid",
  "title": "string",
  "description": "string",
  "url": "string",
  "publication_date": "2025-03-26T04:46:55",
  "source_name": "string",
  "category": ["string"],
  "relevance_score": 0.0-1.0,
  "latitude": float,
  "longitude": float
}
```

Categories include: `technology`, `business`, `sports`, `world`, `national`, `politics`, `entertainment`, `bollywood`, `cricket`, `IPL`, `health`, `science`, `startup`, `crime`, `education`, `travel`, `finance`, `DEFENCE`, etc.

## API Endpoints to Implement

Base URL: `/api/v1/news`

| Endpoint | Method | Query Params | Ranking |
|---|---|---|---|
| `/category` | GET | `category`, `limit` | publication_date desc |
| `/score` | GET | `threshold` (default 0.7), `limit` | relevance_score desc |
| `/search` | GET | `query`, `limit` | combined relevance_score + text match |
| `/source` | GET | `source`, `limit` | publication_date desc |
| `/nearby` | GET | `lat`, `lon`, `radius` (km), `limit` | distance asc (Haversine) |
| `/trending` | GET | `lat`, `lon`, `limit` | computed trending score (bonus) |

All endpoints accept `limit` defaulting to 5 and return top 5 articles.

## LLM Integration

The `/search` endpoint (and a general `/query` endpoint) should:
1. Send the user's natural-language query to an LLM
2. Extract: entities (people, orgs, locations), intent (`category`/`score`/`search`/`source`/`nearby`)
3. Route to the appropriate retrieval strategy
4. Enrich results: each article gets an `llm_summary` field generated by the LLM

Example LLM input/output:
- Query: `"Latest Elon Musk Twitter news near Palo Alto"` → entities: `["Elon Musk","Twitter","Palo Alto"]`, intent: `"nearby"`
- Query: `"Top tech news from New York Times"` → entities: `["New York Times"]`, intent: `["category","source"]`

## Architecture Notes

- **Database**: Load `news_data.json` into a DB on startup. SQL (PostgreSQL with PostGIS for geo queries) or NoSQL (MongoDB with 2dsphere index) both work. For nearby/trending, geospatial indexing is important.
- **Trending feed**: Requires a simulated user-event stream (views/clicks with user_id, article_id, lat/lon, timestamp). Trending score = f(event_count, event_type_weight, recency_decay, geo_proximity). Cache trending results by location cluster (e.g., geohash) using Redis.
- **Caching**: Redis for trending feed; consider caching LLM responses by query hash.
- **LLM API**: Use Claude (`claude-sonnet-4-6`) or OpenAI. Structure the prompt to return JSON with `entities`, `intent`, and optionally `search_keywords`.

## Data Ingestion

The ingest step should read `news_data.json` and bulk-insert into the database. This is typically a one-time migration script run at startup or via a CLI flag.
